Master keys are the foundation of Stackpanel secrets management. They are AGE encryption keys used to encrypt and decrypt all secrets in your project. Instead of per-user keys, a small number of master keys are shared across environments.

## Local Master Key

Every Stackpanel project automatically generates a local master key on first shell entry. This key is stored at:

```
.stackpanel/state/keys/local.txt    # Private key (gitignored)
.stackpanel/state/keys/local.pub    # Public key
```

The local key enables zero-setup secrets. You can start encrypting and decrypting immediately without any external service or team coordination.

<Callout type="info">
The local master key is gitignored because it contains a private key specific to your machine. Every developer gets their own local key automatically.
</Callout>

## Team Master Keys

For secrets that need to be shared across your team, add master keys backed by external secret stores:

```nix
stackpanel.secrets.master-keys = {
  # Local key (auto-configured)
  local = {};

  # Team dev key in AWS SSM
  dev = {
    age-pub = "age1ql3z7hjy54pw3hyww5ayyfg7zqgvc7w3j2elw8zmrj2kg5sfn9aqmcac8p";
    ref = "ref+awsssm://stackpanel/keys/dev";
  };

  # Production key in AWS SSM
  prod = {
    age-pub = "age1abc123...";
    ref = "ref+awsssm://stackpanel/keys/prod";
  };
};
```

Each master key has:

| Field | Description |
|-------|-------------|
| age-pub | The AGE public key (used for encryption) |
| ref | A vals reference to resolve the private key at runtime |
| resolve-cmd | Alternative: a shell command to retrieve the private key |

## Supported Secret Stores

The ref field uses [vals](https://github.com/helmfile/vals) syntax to resolve private keys from external stores:

| Store | Reference Format |
|-------|-----------------|
| AWS SSM Parameter Store | ref+awsssm://path/to/key |
| HashiCorp Vault | ref+vault://secret/data/path#key |
| GCP Secret Manager | ref+gcpsecrets://project/name |
| Azure Key Vault | ref+azurekeyvault://vault/key |
| Local file | ref+file:///path/to/key.txt |

For stores not supported by vals, use resolve-cmd:

```nix
master-keys.prod = {
  age-pub = "age1...";
  resolve-cmd = "op read op://vault/stackpanel/age-key";
};
```

## Multi-Key Encryption

Secrets can be encrypted to multiple master keys simultaneously. This lets you have secrets that both the dev and prod keys can decrypt:

```nix
stackpanel.variables."/shared/openai-key" = {
  key = "OPENAI_API_KEY";
  type = "SECRET";
  master-keys = [ "dev" "prod" ];
};
```

Under the hood, AGE supports encrypting to multiple recipients. The encrypted file can be decrypted by any of the specified keys.

## Generating a Team Master Key

To create a new master key for your team:

```bash
# Generate a new AGE key pair
age-keygen -o team-dev-key.txt

# The public key is printed to stdout, e.g.:
# Public key: age1ql3z7hjy54pw3hyww5ayyfg7zqgvc7w3j2elw8zmrj2kg5sfn9aqmcac8p

# Store the private key in your secret store
aws ssm put-parameter   --name "/stackpanel/keys/dev"   --value ""   --type SecureString

# Add the public key to your Stackpanel config
# (the private key reference points to SSM)
```

Then add it to your config as shown in the Team Master Keys section above. Delete the local copy of the private key after storing it in your secret store.

## Key Rotation

To rotate a master key:

1. Generate a new key pair
2. Store the new private key in your secret store
3. Update the age-pub and ref in your config
4. Re-encrypt all secrets that used the old key

```bash
# Re-encrypt secrets to use the new key
stackpanel vars rekey --old-key dev --new-key dev
```
