Stackpanel stores secrets in SOPS-encrypted YAML files such as `vars/dev.sops.yaml` and `vars/prod.sops.yaml`. Group names are just file buckets now. There is no extra Stackpanel-managed per-group private key layer.

## Getting Started

1. Enter the devshell with `nix develop --impure`.
2. Add your public key in `stackpanel.secrets.recipients` or `stackpanel.users`.
3. Set secrets with `secrets:set KEY --group dev --value ...`.
4. Read them back with `secrets:get KEY --group dev`.

## How It Works

- Stackpanel generates a local AGE key for you on shell entry.
- Recipient public keys live in Nix config.
- `.stack/secrets/.sops.yaml` is generated from the current recipients on each shell entry.
- SOPS encrypts each `vars/*.sops.yaml` file directly to those recipients.
- When recipient keys change, an existing recipient runs `.stack/secrets/bin/rekey.sh` and commits the updated SOPS files.

## Common Commands

| Command | Description |
|---------|-------------|
| `secrets:set KEY --group GROUP --value VALUE` | Set a secret in a group file |
| `secrets:get KEY --group GROUP` | Read a secret from a group file |
| `secrets:list [GROUP]` | List secret keys by group |
| `secrets:load GROUP [--format dotenv\|json\|yaml]` | Decrypt a group's SOPS file |

## Team Workflow

1. A teammate adds their public key in Nix config.
2. They commit the updated config and generated `.stack/secrets/.sops.yaml`.
3. Someone who already has access runs `.stack/secrets/bin/rekey.sh`.
4. The updated `vars/*.sops.yaml` files are committed and shared.
