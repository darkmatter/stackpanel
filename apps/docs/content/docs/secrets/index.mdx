Stackpanel provides a master key-based secrets management system built on [AGE](https://github.com/FiloSottile/age) encryption. Instead of managing per-user encryption keys, your team shares a small number of master keys that encrypt all secrets. Each secret specifies which master keys can decrypt it, giving you fine-grained access control.

## The Three-Tier System

Stackpanel secrets architecture has three layers:

| Tier | What It Is | Where It Lives |
|------|-----------|----------------|
| **Master keys** | AGE keys used to encrypt/decrypt secrets | Local auto-generated + team-shared (AWS SSM, Vault, etc.) |
| **Variables** | Named secret values with type and scope | config.nix (metadata) + .age files (encrypted values) |
| **Resolution** | Runtime decryption of secrets into env vars | The Stackpanel agent at shell entry time |

## Quick Start

Every Stackpanel project starts with a **local master key** that is auto-generated on first shell entry. This means you can use secrets immediately with no external setup required:

```bash
# Set a secret (encrypted with the local master key)
stackpanel vars set API_KEY --value "sk-abc123"

# Get a secret
stackpanel vars get API_KEY

# List all secrets
stackpanel vars list
```

For team-shared secrets, add additional master keys backed by your preferred secret store.

## How It Works

When you set a secret:

1. Look up the master key public AGE key
2. Encrypt the value with age
3. Store the encrypted file in the secrets directory

When decrypting at runtime:

1. The agent tries each configured master key
2. Resolves the private key from its backing store (file, AWS SSM, Vault, etc.)
3. Decrypts the value and exports it as an environment variable

## CLI Commands

| Command | Description |
|---------|-------------|
| stackpanel vars set | Set a secret or variable |
| stackpanel vars get | Get a decrypted value |
| stackpanel vars list | List all variables |
| stackpanel vars delete | Delete a variable |

See [CLI Vars](/docs/cli/vars) for the full command reference.

## External Secret Stores

Master key private keys can be stored in any backend that [vals](https://github.com/helmfile/vals) supports:

| Store | Reference Format |
|-------|-----------------|
| AWS SSM Parameter Store | ref+awsssm://path/to/key |
| HashiCorp Vault | ref+vault://secret/data/path#key |
| GCP Secret Manager | ref+gcpsecrets://project/secret-name |
| Local file | ref+file:///path/to/key.txt |
