This ensures schemas stay in sync with the actual option types.
Schemas are used for YAML file validation in IDEs.

YAML config files (alternative to Nix):
```nix
- config.yaml -> global secrets config
- users.yaml -> team members with AGE keys
- apps/*/config.yaml -> per-app codegen config
- apps/*/common.yaml -> shared secret schema
- apps/*/{dev,staging,prod}.yaml -> environment-specific
```
