Group-based secrets management for StackPanel projects using direct SOPS decryption.

## Overview

This module provides encrypted secrets management using [AGE](https://github.com/FiloSottile/age) encryption with group-based access control. Each group (e.g., `dev`, `prod`) has its own AGE keypair. Secret values are stored directly in SOPS-encrypted YAML files -- variable IDs encode the group and key name.

## Key Concepts

### Groups

Groups are the primary access control boundary. Each group has:
- An AGE keypair (public key in `config.nix`, private key in `.enc.age`)
- A SOPS-encrypted YAML file (`vars/<group>.sops.yaml`)
- Specific recipients who can decrypt

```nix
stackpanel.secrets.groups = {
  dev = { age-pub = "age1..."; };
  prod = { age-pub = "age1..."; };
};
```

### Variable ID Convention

Variable IDs encode the group: `/<group>/<key>`

| Variable ID | Group | SOPS File | Key |
|---|---|---|---|
| `/dev/database-url` | dev | `vars/dev.sops.yaml` | `database-url` |
| `/prod/api-key` | prod | `vars/prod.sops.yaml` | `api-key` |
| `/common/log-level` | common | `vars/common.sops.yaml` | `log-level` |
| `/var/app-name` | var | N/A (literal) | Literal value |
| `/computed/services/pg-port` | computed | N/A | Computed value |

### Variable Types

Type is determined by the ID prefix (keygroup):

| Keygroup | Type | Value Field | Resolved At |
|---|---|---|---|
| `/var/*` | Config (literal) | The actual value | Eval time |
| `/dev/*`, `/prod/*`, `/common/*` | Secret | Empty string `""` | Runtime |
| `/computed/*` | Computed | Derived value | Eval time |

### Example Configuration

```nix
{
  stackpanel.secrets.groups = {
    dev = { age-pub = "age1..."; };
    prod = { age-pub = "age1..."; };
  };

  stackpanel.variables = {
    "/var/app-name" = {
      key = "APP_NAME";
      value = "myapp";
    };

    "/dev/database-url" = {
      key = "DATABASE_URL";
      value = "";  # actual value in vars/dev.sops.yaml
    };

    "/prod/api-key" = {
      key = "API_KEY";
      value = "";  # actual value in vars/prod.sops.yaml
    };
  };
}
```

## Available Commands

| Command | Description |
|---------|-------------|
| `secrets:set KEY --group GROUP --value VALUE` | Set a secret in a SOPS group file |
| `secrets:get KEY --group GROUP` | Decrypt and retrieve a secret |
| `secrets:list [GROUP]` | List all secrets (optionally by group) |
| `secrets:load GROUP [--format dotenv\|json\|yaml]` | Decrypt an entire group file |
| `secrets:join [--group GROUP]` | Add your local key to a recipient group |
| `secrets:init-group NAME` | Initialize a group keypair |
| `secrets:rekey ID --keys k1,k2` | Re-encrypt to different keys |
| `secrets:show-keys` | Show configured groups and keys |

## How It Works

### Encryption (setting a secret)

When you run `secrets:set MY_KEY --group dev --value "secret"`:

1. Look up `groups.dev.age-pub` from `config.nix`
2. Run `sops set vars/dev.sops.yaml '["MY_KEY"]' '"secret"'`
3. SOPS encrypts the value in-place using the group's AGE public key

### Decryption (getting a secret)

When the agent reads `/dev/MY_KEY`:

1. Parse the variable ID: group=`dev`, key=`MY_KEY`
2. Run `sops --decrypt vars/dev.sops.yaml` using `SOPS_AGE_KEY_CMD`
3. `SOPS_AGE_KEY_CMD` resolves keys from: local key, `.age` files, `.enc.age` files
4. Extract `MY_KEY` from the decrypted YAML

### Key Resolution Chain

```
SOPS_AGE_KEY_CMD resolves keys in order:
  1. Local AGE key (.stackpanel/state/keys/local.txt)
  2. Plaintext group keys (recipients/<group>.age, gitignored)
  3. .vals files (recipients/<group>.vals, optional)
  4. SOPS-encrypted group keys (recipients/<group>.enc.age)
```

### Local Development

A `local` AGE key is auto-generated on first shell entry:
- Private key: `.stackpanel/state/keys/local.txt`
- Public key: `.stackpanel/state/keys/local.pub`

Users register as recipients by running `secrets:join`, which copies their public key to `recipients/team/`.

## Infrastructure Output Integration

The infra module system can write outputs directly to SOPS group files:

```nix
stackpanel.infra.storage-backend = {
  type = "sops";
  sops.group = "dev";  # writes to vars/dev.sops.yaml
};
```

`Infra.syncAll()` uses `sops set` per key for non-destructive updates.

## Standalone Scripts (`bin/`)

The `.stackpanel/secrets/bin/` directory contains portable shell scripts that operate without the Nix devshell. These are the underlying operations that the devshell `secrets:*` commands wrap.

| Script | Description |
|--------|-------------|
| `rekey.sh [group]` | Re-encrypt `.enc.age` files to all current recipients. Collects all `.age.pub` and `.ed25519.pub` keys, regenerates `recipients/.sops.yaml`, and re-encrypts each group's `.enc.age`. |
| `rotate.sh <group>` | Full key rotation: archives old key to `recipients/.archive/`, generates new AGE keypair, encrypts new private key as `.enc.age`, re-encrypts `vars/<group>.sops.yaml` to new key. |
| `decrypt.sh <group>\|--all` | Decrypt `.enc.age` to plaintext `.age` (gitignored). Skips already-decrypted groups. |
| `add-recipient.sh --key <key> --name <name> [--group <group>]` | Detects key type (AGE or SSH ED25519), writes `.pub` file to the recipient group directory. |
| `codegen.sh [--app <name>]` | Generates typed TypeScript env modules from `apps.json` (per-app codegen settings) and `vars/*.sops.yaml` (secret key names). Reads `groups.json` for app-to-group access mapping. |

### `codegen.sh` details

This script generates type-safe environment access code without requiring Nix evaluation. It reads:

- `apps.json` -- per-app codegen settings (`language`, `path`)
- `groups.json` -- maps recipient groups to keygroups and apps
- `vars/*.sops.yaml` -- extracts key names (requires decryption)

For each app, it determines which keygroups are accessible (via `groups.json` app membership or all groups as fallback), then generates a TypeScript module with Zod-validated `getEnv()` that covers all keys from those groups.

## External Secret Stores

Group private keys can optionally be resolved from external stores via `.vals` files or the `key-cmd` option:

```nix
# Via key-cmd
stackpanel.secrets.groups.prod.key-cmd = "op read 'op://vault/stackpanel/age-key'";
```

```
# Via .vals file (recipients/prod.vals)
ref+awsssm:///stackpanel/keys/prod
```

Note: `vals` is only used for group key resolution, not for individual secret values. Secret values are decrypted directly from SOPS files.
