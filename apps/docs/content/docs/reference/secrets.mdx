# Secrets Options

<NixOption
  path={"secrets.all-public-keys"}
  type={"list of string"}
  readonly
  module={"secrets"}
>
All configured SOPS recipient public keys after applying Nix-based fallbacks.
</NixOption>

<NixOption
  path={"secrets.codegen"}
  type={"attribute set of (submodule)"}
  defaultValue={"{ }"}
  module={"secrets"}
>
Code generation targets keyed by name (e.g., typescript, go, python).
Used to drive language-specific env/secret helpers.
</NixOption>

<NixOption
  path={"secrets.codegen.<name>.directory"}
  type={"null or string"}
  defaultValue={"null"}
  module={"secrets"}
>
Output directory for generated code (repo-relative)
</NixOption>

<NixOption
  path={"secrets.codegen.<name>.language"}
  type={"null or string"}
  defaultValue={"null"}
  module={"secrets"}
>
Target language for generated code (e.g., "typescript", "go", "python").
Informational only for now; codegen selection is based on the target key.
</NixOption>

<NixOption
  path={"secrets.codegen.<name>.name"}
  type={"null or string"}
  defaultValue={"null"}
  module={"secrets"}
>
Name of the generated package/module (defaults to the target key)
</NixOption>

<NixOption
  path={"secrets.creation-rules"}
  type={"list of (submodule)"}
  defaultValue={"[ ]"}
  exampleCode={"[\n  {\n    path-regex = \"^dev/web\\\\.sops\\\\.yaml$\";\n    recipient-groups = [ \"dev-team\" ];\n  }\n  ]"}
  module={"secrets"}
>
SOPS creation rules rendered into `.stack/secrets/.sops.yaml`.

These rules mirror SOPS directly and can reference both direct
recipients and reusable recipient groups.
</NixOption>

<NixOption
  path={"secrets.creation-rules.*.path-regex"}
  type={"string"}
  exampleCode={"\"^dev/web\\\\.sops\\\\.yaml$\""}
  module={"secrets"}
>
Regex matched against SOPS file paths.
</NixOption>

<NixOption
  path={"secrets.creation-rules.*.recipient-groups"}
  type={"list of string"}
  defaultValue={"[ ]"}
  exampleCode={"[\n  \"dev-team\"\n  \"ci\"\n]"}
  module={"secrets"}
>
Named recipient groups expanded into this rule.
</NixOption>

<NixOption
  path={"secrets.creation-rules.*.recipients"}
  type={"list of string"}
  defaultValue={"[ ]"}
  exampleCode={"[\n  \"alice\"\n]"}
  module={"secrets"}
>
Direct recipient names included in this rule.
</NixOption>

<NixOption
  path={"secrets.creation-rules.*.unencrypted-comment-regex"}
  type={"string"}
  defaultValue={"\"^\\\\s?(safe|plaintext)\""}
  module={"secrets"}
>
Regex for comments that mark plaintext values in SOPS files.
</NixOption>

<NixOption

<NixOption
  path={"secrets.environments.<name>.sources"}
  type={"list of string"}
  defaultValue={"[ ]"}
  module={"secrets"}
>
List of SOPS-encrypted source files for this environment (without .yaml extension).
These files are decrypted and merged to provide secrets for the environment.
</NixOption>

Deprecated legacy groups option.
</NixOption>

<NixOption
  path={"secrets.input-directory"}
  type={"null or string"}
  defaultValue={"null"}
  module={"secrets"}
>
Directory containing SOPS-encrypted secrets (legacy SOPS layout).
Used when decrypting/merging YAML sources defined under environments.
</NixOption>

<NixOption
  path={"secrets.master-keys"}
  type={"attribute set of (submodule)"}
  defaultCode={"{\n  local = {\n    age-pub = \"\";\n    ref = \"ref+file://.stack/keys/local.txt\";\n  };\n}"}
  exampleCode={"{\n  local = {\n    age-pub = \"age1...\";\n    ref = \"ref+file://.stack/keys/local.txt\";\n  };\n  ci = {\n    age-pub = \"age1...\";\n    ref = \"ref+awsssm://stackpanel/keys/ci\";\n  };\n}"}
  module={"secrets"}
>
Master keys for encrypting and decrypting individual `.age` secret files.

These are separate from the SOPS recipient list used for `vars/*.sops.yaml`.
A default local key is always configured so local development works out of the box.
</NixOption>

<NixOption
  path={"secrets.master-keys.<name>.age-pub"}
  type={"string"}
  defaultValue={"\"\""}
  module={"secrets"}
>
AGE public key for encrypting secrets to this key.
Format: age1... (bech32-encoded)
</NixOption>

<NixOption
  path={"secrets.master-keys.<name>.ref"}
  type={"string"}
  defaultValue={"\"\""}
  module={"secrets"}
>
Vals reference that resolves to the AGE private key.
Examples:
  - ref+file://.stack/keys/local.txt (local file)
  - ref+awsssm://stackpanel/keys/dev (AWS SSM Parameter Store)
  - ref+vault://secret/data/stackpanel/prod#key (HashiCorp Vault)
</NixOption>

<NixOption
  path={"secrets.master-keys.<name>.resolve-cmd"}
  type={"null or string"}
  defaultValue={"null"}
  module={"secrets"}
>
Custom command to resolve the private key (overrides ref).
The command should output the AGE private key to stdout.
Example: op read 'op://vault/stackpanel/age-key'
</NixOption>

<NixOption
  path={"secrets.recipient-groups"}
  type={"attribute set of (submodule)"}
  defaultValue={"{ }"}
  exampleCode={"{\n  dev-team.recipients = [ \"alice\" \"bob\" ];\n  ci.recipients = [ \"buildkite\" ];\n}"}
  module={"secrets"}
>
Reusable recipient sets that can be referenced by SOPS creation rules.
</NixOption>

<NixOption
  path={"secrets.recipient-groups.<name>.recipients"}
  type={"list of string"}
  defaultValue={"[ ]"}
  exampleCode={"[\n  \"alice\"\n  \"buildkite\"\n]"}
  module={"secrets"}
>
Named recipients included by this reusable recipient group.
</NixOption>

<NixOption
  path={"secrets.recipients"}
  type={"attribute set of (submodule)"}
  defaultValue={"{ }"}
  exampleCode={"{\n  cooper = {\n    public-key = \"age1psa52j93p0t7rej4lyzeww6hzg9hh4ylxu6v30tcgag44apw8als2xg3ef\";\n    tags = [ \"dev\" \"prod\" \"shared\" ];\n  };\n  coop_mac_studio = {\n    public-key = \"ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIFauRe+VXvSmsca73hmxrylRPiueX/aHbXUu1jz7kGB8\";\n    tags = [ \"dev\" ];\n  };\n}"}
  module={"secrets"}
>
SOPS recipients declared in Nix.

These entries are rendered into `.stack/secrets/.sops.yaml`.
If left empty, Stackpanel falls back to recipients derived from
`stackpanel.users.*.public-keys` and `secrets-allowed-environments`.
</NixOption>

<NixOption
  path={"secrets.recipients.<name>.public-key"}
  type={"string"}
  exampleCode={"\"age1psa52j93p0t7rej4lyzeww6hzg9hh4ylxu6v30tcgag44apw8als2xg3ef\""}
  module={"secrets"}
>
Public key to include in the generated `.stack/secrets/.sops.yaml`.

Supports both AGE (`age1...`) and SSH Ed25519 (`ssh-ed25519 ...`) recipients.
</NixOption>

<NixOption
  path={"secrets.recipients.<name>.tags"}
  type={"list of string"}
  defaultValue={"[ ]"}
  exampleCode={"[\n  \"dev\"\n  \"shared\"\n]"}
  module={"secrets"}
>
Tags used to select this recipient for secret groups.

A secret group includes every recipient whose tags overlap with the
group's configured tags.
</NixOption>

<NixOption
  path={"secrets.secrets-dir"}
  type={"null or string"}
  defaultValue={"null"}
  module={"secrets"}
>
Directory where secret .age files are stored (default: .stack/secrets)
</NixOption>

<NixOption
  path={"secrets.sops-age-keys.op-refs"}
  type={"list of string"}
  defaultValue={"[ ]"}
  exampleCode={"[\n  \"op://travel/age-keys/local/private\"\n  \"op://travel/age-keys/shared/private\"\n]"}
  module={"secrets"}
>
Optional `op://` references queried with `op read`.

If configured, `sops-age-keys` attempts these references after
configured file paths.
</NixOption>

<NixOption
  path={"secrets.sops-age-keys.paths"}
  type={"list of string"}
  defaultValue={"[ ]"}
  exampleCode={"[\n  \"/tmp/team-keys/sops.age\"\n  \"/run/secrets/ci.age\"\n]"}
  module={"secrets"}
>
Additional private key file paths searched by `sops-age-keys` after
`user-key-path` and `repo-key-path`.

Paths are tested with `[[ -f ]]` as provided, so they can be absolute
or relative to your current working directory.
</NixOption>

<NixOption
  path={"secrets.sops-age-keys.repo-key-path"}
  type={"null or string"}
  defaultValue={"\".stack/keys/local.txt\""}
  exampleCode={"\"\\\".stack/keys/local.txt\\\"\""}
  module={"secrets"}
>
Repo-local fallback AGE key path generated by Stackpanel for development.

This is convenient for bootstrapping, but a user-level or external key
source is preferred for long-term use.
</NixOption>

<NixOption
  path={"secrets.sops-age-keys.sources"}
  type={"list of (submodule)"}
  defaultCode={"[\n  {\n    name = \"User key path\";\n    type = \"user-key-path\";\n    value = \"$HOME/Library/Application Support/sops/age/keys.txt\";\n  }\n  {\n    name = \"Repo key path\";\n    type = \"repo-key-path\";\n    value = \".stack/keys/local.txt\";\n  }\n]"}
  module={"secrets"}
>
Ordered key sources tried by `sops-age-keys`.

This is the preferred configuration model for the UI. File-like
sources are tried in order until one yields an AGE private key.
</NixOption>

<NixOption
  path={"secrets.sops-age-keys.user-key-path"}
  type={"null or string"}
  defaultValue={"\"$HOME/Library/Application Support/sops/age/keys.txt\""}
  exampleCode={"\"\\\"$HOME/Library/Application Support/sops/age/keys.txt\\\"\""}
  module={"secrets"}
>
Primary user-level AGE key path for SOPS.

This should point to a user-managed key file outside the repo.
Stackpanel checks this before any additional custom paths.
</NixOption>

<NixOption
  path={"secrets.system-keys"}

>
System-level AGE public keys (CI, deploy servers, etc.).
These keys can decrypt all secrets regardless of environment restrictions.
</NixOption>
