# Secrets Options

## `secrets.codegen`

Code generation settings per target

| Property | Value |
|----------|-------|
| **Type** | `attribute set of (submodule)` |
| **Default** | `{ }` |

---

## `secrets.codegen.<name>.directory`

Output directory for generated code (relative to project root)

| Property | Value |
|----------|-------|
| **Type** | `string` |
| **Default** | `""` |

---

## `secrets.codegen.<name>.language`

Programming language for generated code

| Property | Value |
|----------|-------|
| **Type** | `attribute set` |
| **Default** | `{ }` |

---

## `secrets.codegen.<name>.name`

Name of the generated code package

| Property | Value |
|----------|-------|
| **Type** | `string` |
| **Default** | `""` |

---

## `secrets.enable`

Enable secrets management

| Property | Value |
|----------|-------|
| **Type** | `boolean` |
| **Default** | `false` |

---

## `secrets.environments`

Environment-specific secrets configurations

| Property | Value |
|----------|-------|
| **Type** | `attribute set of (submodule)` |
| **Default** | `{ }` |

---

## `secrets.environments.<name>.name`

Name of the environment (e.g., 'production', 'staging')

| Property | Value |
|----------|-------|
| **Type** | `string` |
| **Default** | `""` |

---

## `secrets.environments.<name>.public-keys`

AGE public keys that can decrypt secrets for this environment

| Property | Value |
|----------|-------|
| **Type** | `list of string` |
| **Default** | `[ ]` |

---

## `secrets.environments.<name>.sources`

List of SOPS-encrypted source files for this environment (without .yaml extension)

| Property | Value |
|----------|-------|
| **Type** | `list of string` |
| **Default** | `[ ]` |

---

## `secrets.input-directory`

Directory where SOPS-encrypted secrets are stored

| Property | Value |
|----------|-------|
| **Type** | `string` |
| **Default** | `""` |

---

## `secrets.wrapped.enable`

Enable creation of wrapped packages that inject secrets as environment variables.

| Property | Value |
|----------|-------|
| **Type** | `boolean` |
| **Default** | `false` |

---

## `secrets.wrapped.projectRoot`

Path to the project root directory. Used to locate secrets files at runtime. Can be overridden by setting `PROJECT_ROOT` environment variable.

| Property | Value |
|----------|-------|
| **Type** | `string` |
| **Default** | `"."` |

---

## `secrets.wrapped.apps`

Per-app configuration for wrapped packages. Each app can specify which packages to wrap and for which environments.

| Property | Value |
|----------|-------|
| **Type** | `attribute set of (submodule)` |
| **Default** | `{ }` |

### Example

```nix
stackpanel.secrets.wrapped.apps = {
  myapp = {
    packages = [ pkgs.nodejs pkgs.python3 ];
    environments = [ "dev" "staging" "prod" ];
  };
};
```

---

## `secrets.wrapped.apps.<name>.packages`

List of packages to create wrapped versions of. Each package will have wrapped versions created for each environment.

| Property | Value |
|----------|-------|
| **Type** | `list of package` |
| **Default** | `[ ]` |

---

## `secrets.wrapped.apps.<name>.environments`

List of environments to create wrapped packages for.

| Property | Value |
|----------|-------|
| **Type** | `list of string` |
| **Default** | `[ "dev" "staging" "prod" ]` |

---

## `secrets.wrapped.packages`

Generated wrapped packages, organized by app name, environment, and package name. This is a read-only attribute.

| Property | Value |
|----------|-------|
| **Type** | `attribute set of (attribute set of (attribute set of package))` |
| **Default** | `{ }` |

### Usage

Access wrapped packages via:

```nix
config.stackpanel.secrets.wrapped.packages.<app>.<env>.<package>

# Example:
config.stackpanel.secrets.wrapped.packages.myapp.dev.nodejs
config.stackpanel.secrets.wrapped.packages.myapp.prod.python3
```

---

## `secrets.wrapped.addToDevshell`

Whether to add wrapped packages to the devshell. Usually you only want specific wrapped packages, so this defaults to false.

| Property | Value |
|----------|-------|
| **Type** | `boolean` |
| **Default** | `false` |

---

## `secrets.wrapped.devshellEnvironment`

When `addToDevshell` is true, which environment's wrapped packages to add. Set to `null` to add all environments (not recommended).

| Property | Value |
|----------|-------|
| **Type** | `null or string` |
| **Default** | `"dev"` |
