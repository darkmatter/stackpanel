# Users Options

## `users`

Team members with project access.

Note: Secrets are encrypted using master keys, not per-user keys.
See stackpanel.secrets.master-keys for encryption configuration.

| Property | Value |
|----------|-------|
| **Type** | `attribute set of (submodule)` |
| **Default** | `{ }` |

**Example:**

```nix
{
  johndoe = {
    name = "John Doe";
    github = "johndoe";
    email = "john@example.com";
  };
  alice = {
    name = "Alice Example";
    github = "aliceexample";
  };
}
```


---

Email address

| Property | Value |
|----------|-------|
| **Type** | `null or string` |

## `users.<name>.github`

GitHub username

| Property | Value |
|----------|-------|
| **Type** | `null or string` |
| **Default** | `null` |

---

## `users.<name>.name`

Display name of the user

| Property | Value |
|----------|-------|
| **Type** | `string` |
| **Default** | `""` |

---

## `users.<name>.public-keys`

SSH or AGE public keys for the user

| Property | Value |
|----------|-------|
| **Type** | `list of string` |
| **Default** | `[ ]` |

---

## `users.<name>.secrets-allowed-environments`

Environments this user can access secrets for (e.g., dev, staging, production)

| Property | Value |
|----------|-------|
| **Type** | `list of string` |
| **Default** | `[ ]` |
