# Services Options

<NixOption
  path={"serviceModules"}
  type={"list of module"}
  defaultValue={"[ ]"}
  module={"services"}
>
Additional modules to extend service configuration options.

This allows other modules (like process-compose) to inject per-service
options into every service definition. Works identically to appModules.

Example: The process-compose module injects process-compose.namespace,
process-compose.readiness_probe, etc. into every service.
</NixOption>

<NixOption
  path={"services"}
  type={"attribute set of (submodule)"}
  defaultValue={"{ }"}
  exampleCode={"{\n  postgres = {\n    enable = true;\n    displayName = \"PostgreSQL\";\n    command = \"${postgresStartScript}/bin/postgres-start\";\n    port = 5432;\n    autoStart = true;\n    env = {\n      DATABASE_URL = \"postgresql://localhost:5432/mydb\";\n    };\n    packages = [ pkgs.postgresql_17 ];\n  };\n  redis = {\n    enable = true;\n    displayName = \"Redis\";\n    command = \"${redisStartScript}/bin/redis-start\";\n    port = 6379;\n  };\n}"}
  module={"services"}
>
Development services managed by process-compose.

Each service is a long-running process (database, cache, object store, etc.)
that gets a process-compose entry under the "services" namespace.

Services can be defined directly here or via the stackpanel.globalServices
convenience layer (which maps into this option internally).

The process-compose module injects additional per-service options
(namespace, readiness_probe, availability, depends_on) via serviceModules.
</NixOption>

<NixOption
  path={"services.<name>.autoStart"}
  type={"boolean"}
  defaultValue={"true"}
  module={"services"}
>
Whether to auto-start this service when `dev` is run.
If false, the service is visible in the process-compose TUI
but must be started manually.
</NixOption>

<NixOption
  path={"services.<name>.command"}
  type={"string"}
  exampleCode={"\"${postgresStartScript}/bin/postgres-start\""}
  module={"services"}
>
Command to start the service in foreground mode.
This should be an idempotent script that initializes data
directories if needed, then exec's the service binary.
Process-compose manages the lifecycle (restart, shutdown).
</NixOption>

<NixOption
  path={"services.<name>.dataDir"}
  type={"null or string"}
  defaultValue={"null"}
  module={"services"}
>
Path where this service stores its data.
Typically under .stack/state/services//.
</NixOption>

<NixOption
  path={"services.<name>.description"}
  type={"string"}
  defaultValue={"\"\""}
  exampleCode={"\"Relational database for application data\""}
  module={"services"}
>
Short description of the service.
</NixOption>

<NixOption
  path={"services.<name>.displayName"}
  type={"string"}
  defaultValue={"\"‹name›\""}
  exampleCode={"\"PostgreSQL\""}
  module={"services"}
>
Human-readable display name for the service.
</NixOption>

<NixOption
  path={"services.<name>.enable"}
  type={"boolean"}
  defaultValue={"false"}
  exampleCode={"true"}
  module={"services"}
>
Whether to enable this service.
</NixOption>

<NixOption
  path={"services.<name>.env"}
  type={"attribute set of string"}
  defaultValue={"{ }"}
  exampleCode={"{\n  DATABASE_URL = \"postgresql://localhost:5432/mydb\";\n  PGHOST = \"/tmp/postgres\";\n}"}
  module={"services"}
>
Environment variables contributed by this service to the devshell.
These are available to all processes and the interactive shell.
</NixOption>

<NixOption
  path={"services.<name>.packages"}
  type={"list of package"}
  defaultValue={"[ ]"}
  module={"services"}
>
Packages added to the devshell PATH when this service is enabled.
Typically includes the service binary and CLI tools (psql, redis-cli, etc.).
</NixOption>

<NixOption
  path={"services.<name>.port"}
  type={"null or 16 bit unsigned integer; between 0 and 65535 (both inclusive)"}
  defaultValue={"null"}
  exampleCode={"5432"}
  module={"services"}
>
Primary port for this service. Optional - some services
(like file watchers) don't use ports.
</NixOption>

<NixOption
  path={"services.<name>.shellHook"}
  type={"strings concatenated with \"\\n\""}
  defaultValue={"\"\""}
  module={"services"}
>
Shell hook code run during devshell initialization.
Used for environment setup that can't be expressed as static env vars.
</NixOption>
