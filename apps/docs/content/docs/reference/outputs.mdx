# Outputs Options

<NixOption
  path={"flakeApps"}
  type={"attribute set of (submodule)"}
  defaultValue={"{ }"}
  module={"outputs"}
>
Flake apps to expose via `nix run .#`.

Each app must have:
  - type: "app"
  - program: Path to executable (usually from a derivation)

Example:
  stackpanel.flakeApps = \{
    web = \{ type = "app"; program = "$\{myPackage\}/bin/web"; \};
  \};
</NixOption>

<NixOption
  path={"flakeApps.<name>.program"}
  type={"string"}
  module={"outputs"}
>
Path to the executable.
</NixOption>

<NixOption
  path={"flakeApps.<name>.type"}
  type={"string"}
  defaultValue={"\"app\""}
  module={"outputs"}
>
App type (always 'app' for Nix apps).
</NixOption>

<NixOption
  path={"outputs"}
  type={"attribute set of (package or attribute set of package)"}
  defaultValue={"{ }"}
  exampleCode={"{\n  docs-json = optionsDoc.optionsJSON;\n  docs-md = optionsDoc.optionsCommonMark;\n  scripts = { my-script = pkgs.writeShellScriptBin \"test\" \"echo hello\"; };\n}"}
  module={"outputs"}
>
Output derivations to expose.

These will be available as:
  - `nix build .#` (via flake) - for direct packages
  - `nix run .#.` (via flake) - for grouped packages (like scripts)
  - `devenv outputs.` (via devenv)

Example:
  stackpanel.outputs = \{
    my-docs = pkgs.runCommand "docs" \{\} "...";
    my-script = pkgs.writeShellScriptBin "foo" "...";
    # Grouped packages (available via nix run .#scripts.)
    scripts = \{ foo = pkgs.writeShellScriptBin "foo" "..."; \};
  \};
</NixOption>
