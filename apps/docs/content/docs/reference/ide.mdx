# Ide Options

<NixOption
  path={"ide.cursor.enable"}
  type={"boolean"}
  defaultValue={"false"}
  exampleCode={"true"}
  module={"ide"}
>
Whether to enable Cursor editor integration (not yet implemented).
</NixOption>

<NixOption
  path={"ide.enable"}
  type={"boolean"}
  defaultValue={"false"}
  exampleCode={"true"}
  module={"ide"}
>
Generate IDE configuration files into .stack/gen/ide/
</NixOption>

<NixOption
  path={"ide.vscode.enable"}
  type={"boolean"}
  defaultValue={"false"}
  exampleCode={"true"}
  module={"ide"}
>
Generate VS Code workspace and configuration files
</NixOption>

<NixOption
  path={"ide.vscode.existing-settings-path"}
  type={"null or absolute path"}
  defaultValue={"null"}
  module={"ide"}
>
Path to existing VS Code settings.json to merge with generated settings.

WARNING: This creates IMPURE evaluation - the file must exist at Nix evaluation time.
</NixOption>

<NixOption
  path={"ide.vscode.extensions"}
  type={"list of string"}
  defaultValue={"[ ]"}
  module={"ide"}
>
Recommended VS Code extension IDs
</NixOption>

<NixOption
  path={"ide.vscode.extra-folders"}
  type={"list of attribute set of string"}
  defaultValue={"[ ]"}
  module={"ide"}
>
Additional workspace folders to include
</NixOption>

<NixOption
  path={"ide.vscode.output-mode"}
  type={"one of \"workspace\", \"settingsJson\""}
  defaultValue={"\"workspace\""}
  module={"ide"}
>
Where to output VS Code settings:
- "workspace": Generate .stack/gen/ide/vscode/stackpanel.code-workspace (default, safe)
- "settingsJson": Generate .vscode/settings.json (CAUTION: may overwrite existing file)
</NixOption>

<NixOption
  path={"ide.vscode.settings"}
  type={"attribute set of anything"}
  defaultValue={"{ }"}
  module={"ide"}
>
VS Code settings to include in the generated configuration.
These take highest priority and will override any existing or generated settings.
</NixOption>

<NixOption
  path={"ide.vscode.workspace-name"}
  type={"string"}
  defaultValue={"\"stackpanel\""}
  module={"ide"}
>
Name for the generated .code-workspace file (without extension)
</NixOption>

<NixOption
  path={"ide.zed.enable"}

  defaultValue={"false"}
  exampleCode={"true"}
  module={"ide"}
>
Generate Zed configuration files
</NixOption>

<NixOption
  path={"ide.zed.existing-settings-path"}

Path to existing Zed settings.json to merge with generated settings.

WARNING: This creates IMPURE evaluation - the file must exist at Nix evaluation time.
</NixOption>

<NixOption
  path={"ide.zed.extensions"}
  type={"list of string"}
  defaultValue={"[ ]"}
  module={"ide"}
>
Recommended Zed extension IDs
</NixOption>

<NixOption
  path={"ide.zed.output-mode"}
  type={"one of \"generated\", \"dotZed\""}
  defaultValue={"\"generated\""}
  module={"ide"}
>
Where to output Zed settings:
- "generated": Generate to .stack/gen/zed/ (default, safe) - requires manual symlink
- "dotZed": Generate to .zed/settings.json (CAUTION: may overwrite existing file)
</NixOption>

<NixOption
  path={"ide.zed.settings"}
  type={"attribute set of anything"}
  defaultValue={"{ }"}
  module={"ide"}
>
Zed settings to include in the generated configuration.
These take highest priority and will override any existing or generated settings.
</NixOption>

<NixOption
  path={"ide.zed.tasks"}
  type={"list of attribute set of anything"}
  defaultValue={"[ ]"}
  exampleCode={"[\n  {\n    args = [\n      \"run\"\n      \"dev\"\n    ];\n    command = \"bun\";\n    label = \"dev\";\n  }\n]"}
  module={"ide"}
>
Zed tasks to include in the generated tasks.json.
Each task is an attrset with label, command, args, etc.
</NixOption>
