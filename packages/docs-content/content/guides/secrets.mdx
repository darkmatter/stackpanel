# Secrets

Secrets are encrypted values stored safely in version control. Stackpanel uses [SOPS](https://github.com/getsops/sops) with [AGE](https://github.com/FiloSottile/age) encryption, organized by groups (e.g., `dev`, `prod`).

## How It Works

1. **Initialize a group** — generates an AGE keypair for the group
2. **Set secrets** — values are SOPS-encrypted in per-group YAML files
3. **Decrypt at runtime** — your local AGE key unlocks group keys, which unlock secrets
4. **Team access** — self-service onboarding via git push + GitHub Actions rekey

## Directory Structure

```
.stack/
├── config.nix              # Group public keys (source of truth)
├── keys/                   # Local AGE keypair (gitignored)
│   ├── local.txt           # Your local AGE private key
│   └── local.pub           # Your local AGE public key
└── secrets/
    ├── vars/               # SOPS-encrypted secret files
    │   ├── dev.sops.yaml
    │   └── prod.sops.yaml
    ├── recipients/         # Team member public keys + encrypted group keys
    │   ├── team/
    │   │   └── alice.age.pub
    │   ├── dev.enc.age     # Group private key (SOPS-encrypted to all recipients)
    │   └── .sops.yaml      # Auto-generated from recipients/**/*.age.pub
    └── bin/                # Standalone scripts (work without Nix)
        ├── rekey.sh
        ├── decrypt.sh
        ├── rotate.sh
        └── add-recipient.sh
```

## Commands

```bash
secrets:init-group dev             # Initialize the dev group
secrets:set API_KEY --group dev    # Set a secret
secrets:get API_KEY --group dev    # Retrieve a secret
secrets:list                       # List all secrets
secrets:join                       # Register your key as a team recipient
secrets:load dev                   # Decrypt a group's SOPS file
```

## Tips

- Your local AGE key is auto-generated on first devshell entry
- Run `secrets:join` to register yourself as a team recipient
- Group public keys go in `.stack/config.nix` after `secrets:init-group`
- Use the standalone scripts in `.stack/secrets/bin/` for CI/CD or non-Nix environments
