# Secrets

Secrets are encrypted values that stay safe in version control. Stackpanel uses AGE encryption with optional AWS KMS support.

## How It Works

1. **Create a secret** — Encrypted with team members' public keys
2. **Store in git** — `.age` files are safe to commit
3. **Decrypt at runtime** — Only authorized users can read values

## Encryption Methods

| Method | Use Case |
|--------|----------|
| **AGE keys** | Personal decryption key (`~/.config/age/key.txt`) |
| **SSH keys** | Use existing SSH key (`~/.ssh/id_ed25519`) |
| **AWS KMS** | CI/CD and production environments |

## Directory Structure

```
.stackpanel/
├── secrets/
│   └── vars/           # Encrypted .age files
│       ├── API_KEY.age
│       └── DB_PASSWORD.age
└── state/              # Local config (gitignored)
    ├── age-identity    # Path to your key
    └── age-key.txt     # Or the key itself
```

## Commands

```bash
generate-sops-config   # Generate .sops.yaml
generate-sops-secrets  # Create environment YAML files
sops edit secrets.yaml # Edit encrypted files
```

## Tips

- Use the Setup wizard to configure your decryption key
- Sync team keys from GitHub collaborators
- Add KMS for CI/CD pipelines that can't use personal keys
